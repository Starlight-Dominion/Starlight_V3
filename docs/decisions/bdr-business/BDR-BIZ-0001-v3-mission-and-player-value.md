# BDR-BIZ-0001: Define v3 Mission and Player Value Priorities

- Status: accepted
- Date: 2026-04-14
- Owner: Product Owner
- Approvers: Technical Lead, Game Design Lead
- Supersedes: none
- Superseded by: none
- Scope: business
- Imported from: old starlight V3 corpus

## Context

v3 needs a clear mission and product direction before deep implementation. In prior versions, intent was implied by features and balancing docs rather than enforced as a foundational source of truth.

## Decision

Define and adopt mission priorities for v3:

1. Strategic depth over raw grind.
2. Fair competition through anti-runaway mechanics.
3. Transparent systems and predictable player outcomes.
4. Sustainable operations and maintainable code.

Mission and charter docs become normative references for roadmap and feature scope.

## Consequences

Positive:

- Product scope decisions become consistent.
- Team can evaluate features against explicit priorities.

Negative:

- Some legacy behaviors may be out of scope and require explicit exceptions.

## Alternatives Considered

- Keep mission implicit in roadmap tickets
- Optimize mainly for rapid feature delivery

## Rollout Plan

1. Publish mission and charter base docs.
2. Require scope checks against mission priorities.
3. Review roadmap items for alignment.

## Validation Plan

- All first-wave docs map to mission priorities.
- New roadmap epics include mission alignment notes.
